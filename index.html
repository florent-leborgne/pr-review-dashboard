Track pull requests you need to review across all repositories, including team-based review requests.

Your token is stored only in your browser's localStorage and is never sent anywhere except directly to the GitHub API.

Which token type should I use?
Fine-grained PAT (recommended)
More secure: scoped to specific repos/orgs, with an expiration date.
Go to Settings → Fine-grained tokens and grant:
• Repository access: All repositories (or select the repos you review)
• Organization permissions: Members → read (to detect your teams)
• Repository permissions: Pull requests → read, Metadata → read
• Set an expiration (e.g. 90 days)

Classic PAT
Simpler to set up, broader access.
Create one with repo and read:org scopes.
